# 촬영 기준 (Capture Guidelines)

> 이 문서는 Notion 회의록·기술 문서에서 확인된 촬영 관련 기록만을 근거로 작성했습니다. 과제 지시문에서 언급된 "벤치프레스 15~30도", "데드리프트 15~20도"처럼 구체적인 각도 수치는 **이번 조사에서 어떤 Notion 페이지에서도 확인되지 않았습니다.** 대신 실제로 문서화된 촬영 원칙만 아래에 정리합니다.

## 1. 공통 원칙 (확인됨)

- 기본 촬영 방향은 **측면(側面)**입니다. "기준지표" 문서 곳곳에서 지표를 "정면영상 표현"과 "측면영상 표현"으로 구분하고 있고, 데이터 경로에 `data/측면/...`가 반복 등장합니다(0611 "yolo 모델 검출").
- 0602 회의록에 "종목별 촬영 규격(카메라 각도) 확정"이라는 기록이 있어 1학기 말에 촬영 규격을 팀 내부적으로 확정한 것으로 보이나, **구체적인 각도 수치가 회의록 본문에서 확인되지 않았습니다** (언급 없음 — 세부 수치는 팀 내부 자료/발표자료 첨부파일에 있을 가능성이 있으나 이번 조사 범위에서는 텍스트로 확인하지 못했습니다).

## 2. 벤치프레스 / 데드리프트: 다각도 촬영 필요성 (확인됨)

정측면 촬영 시 바벨/원판이 손목·정강이 등 주요 관절을 가리는 문제가 반복적으로 지적되었습니다.

> "원판에 의해 바가 가려질 땐 검출이 불가능하다. 따라서 45도 다각도뷰 사용을 해야한다" ("기준지표")

이 문서에서는 **45도**라는 수치가 확인되었으며, 과제 지시문의 "15~30도"/"15~20도"와는 다른 값입니다. 두 수치 중 어느 쪽이 실제로 채택되었는지는 이번 조사만으로 단정할 수 없으므로, 2학기에 팀 내부 확인이 필요합니다.

## 3. 카메라 각도/방향 정규화 (부분 구현 — 종목별로 다름)

`unified_feedback_v4.py` 코드 확인 결과, 카메라 회전 메타데이터(0/90/180/270도, `ffprobe` 기반) 보정은 **구현 완료**입니다. 전문가-사용자 화면의 방향(각도) 정렬은 **벤치프레스만 구현**되어 있고(`ALIGN_EXPERT_ORIENTATION["benchpress"] = True`), 스쿼트·데드리프트는 의도적으로 꺼져 있습니다. 코드 주석에 따르면 데드리프트는 "동작 중 상체가 숙임→직립으로 전신축이 크게 변해, 회전을 고정하면 특정 구간에서 오히려 더 어색해 보이기 때문"입니다. 즉 미구현이 아니라 **"종목 특성상 단순 회전 정렬이 부적절하다고 판단해 끈 것"**입니다 — 자세한 내용은 [`pose_landmarks_and_csv.md`](./pose_landmarks_and_csv.md) 4절 참고.

## 4. 좌우 방향 통일 (구현 완료)

사용자 영상과 전문가 영상의 좌우 방향(카메라를 보는 방향)은 하드코딩된 좌우반전 로직으로 통일하도록 이미 구현되어 있습니다([`pose_landmarks_and_csv.md`](./pose_landmarks_and_csv.md) 참고).

## 5. 가로/세로 영상 (확인 안 됨)

과제 지시문에서 언급된 "가로 영상 사용, 세로 영상 지양"이라는 원칙은 이번 조사에서 명시적으로 확인되지 않았습니다(언급 없음).

## 요약 표

| 항목 | 상태 | 근거 |
|---|---|---|
| 측면 촬영 기본 | 확인됨 | 기준지표, yolo 모델 검출 |
| 특정 각도(15~30도, 15~20도) | **미확인** | 조사 범위 내 없음 |
| 45도 다각도뷰 (바벨 가림 대응) | 확인됨 | 기준지표 |
| 종목별 촬영 규격 확정 여부 | 확정했다는 기록만 확인, 세부 수치 미확인 | 0602 |
| 카메라 각도/회전 정규화 | 향후 적용 예정 | 사용자/기준 영상 정규화 및 판단지표 |
| 좌우 방향 통일 | 구현 완료 | 사용자/기준 영상 정규화 및 판단지표 |
| 가로/세로 영상 통일 | 미확인 | 조사 범위 내 없음 |

**2학기 권장 조치**: 팀이 실제로 사용 중인 촬영 각도(예: 45도 등)를 이 문서에 정식으로 기록하고, 데이터셋에 촬영 각도 메타데이터를 추가하는 것을 권장합니다.
