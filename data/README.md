# data/

이 폴더는 데이터셋의 **구조와 사용 방법을 설명하는 용도**이며, 원본 영상/이미지 파일은 포함하지 않습니다.

## 원본 데이터를 여기에 올리지 않는 이유

- 팀이 촬영한 운동 영상에는 촬영자의 얼굴 등 개인정보가 포함될 수 있습니다.
- 공개 GitHub 저장소에 개인정보가 포함된 원본 영상을 업로드하지 않는 것이 원칙입니다.
- 현재(1학기 종료 시점) 이 저장소에는 실제 데이터 파일이 전혀 없는 상태입니다. 팀이 보유한 원본 데이터의 실제 보관 위치(로컬/Google Drive/Colab 등)는 Notion 문서에서 명시적으로 확인되지 않았습니다.

## 데이터셋 구조 (Notion 기록 기준, 2학기에 실제 폴더 구조로 확정 필요)

Notion 기록상 데이터 경로는 `data/측면/{종목}/...` 형태로 확인됩니다(0611 "yolo 모델 검출"). 2학기에 실제 데이터를 정리할 때는 다음과 같은 구조를 권장합니다.

```text
data/
├─ README.md          (이 파일)
└─ samples/            (개인정보가 없는 짧은 샘플 영상/이미지만, 용량 제한 확인 후 추가)
```

## 샘플 데이터 추가 시 권장 절차

1. 개인 식별 정보(얼굴 등)가 없는지 확인
2. 용량이 큰 원본 대신 짧은 클립 또는 GIF/이미지 캡처로 변환
3. `data/samples/`에 추가하고, 이 README에 파일 목록과 설명을 갱신
4. 여전히 용량이 크다면 Git LFS 도입 여부를 팀과 상의 후 결정 (현재 이 저장소는 Git LFS를 사용하지 않습니다 — 아래 참고)

## Git LFS 관련

이 저장소는 현재 Git LFS를 사용하지 않습니다. 대용량 영상/모델 파일을 추가해야 할 경우, 임의로 LFS를 도입하기보다 다음 대안을 먼저 검토할 것을 권장합니다.

- 대표 결과 영상만 짧은 샘플로 저장
- GIF/이미지 캡처로 변환
- 외부 저장 위치(Google Drive 등)를 문서에 링크만 남기기

팀 논의 후 LFS 도입이 필요하다고 결정되면 이 README와 `.gitignore`를 함께 갱신하십시오.
