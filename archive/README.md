## 현재 상태

이번 문서화 작업 시점에 이 GitHub 저장소에는 실제 코드 파일이 전혀 없었기 때문에(README 2줄짜리 초기 커밋만 존재), **archive로 옮길 파일이 없습니다.**

Notion 기록상으로는 다음과 같이 여러 버전이 존재했던 것으로 보이는 코드들이 있습니다(2학기에 실제 코드를 저장소에 추가할 때 참고).

- `squat_feedback_v3.ipynb` → `v4` → `v5` (0526, 몸통 기울기·발 너비 로직이 단계적으로 업그레이드됨)
- `unified_feedback_v4.py` — 이전 버전(v1~v3)의 존재 여부는 Notion에서 명시적으로 확인되지 않음
- Binary mask 기반 RepCounter v1 → Gaussian density 기반 v2 (0505, v1은 실패로 판단되어 v2로 대체됨)

## 2학기 파일 이관 시 원칙

1. 기존 파일을 바로 삭제하지 않는다.
2. 동일 기능의 여러 버전이 있다면 최신 버전을 우선 `src/`에 배치하고, 이전 버전은 이 폴더로 이동하거나 Git 히스토리에만 남긴다.
3. 파일을 옮기기 전, 어떤 버전이 최신이고 실제로 사용 중인지 팀 내에서 확인한다.
4. 확실하지 않은 파일은 삭제하지 말고 이 폴더로 옮기는 것을 우선 후보로 검토한다.
